Problem Statement

Development of a mission control panel prototype for search and rescue drones in participation with the fire department KampLintfort and research project “Phenix” at HSRW.

With the advent of technological advancements in the field of drones, there has been a growing need to implement drone usage for emergency services. Of interest to us is the Fire Service. Many well-funded Fire Departments now employ search and rescue drones as first responders when dealing with an emergency situation, before sending in their human personnel. The Kamp Lintfort Fire Department does not make this cut, as its a small and underfunded Fire Service that depends mostly on volunteer firefighters. HSRW in partnership with the Kamp Lintfort Fire Department came up with the Phenix Project which aims to create a fully functional search and rescue drone for assessing fire situations. The high point of this project is the development of the drone using cheap and easily purchased parts. Our team is tasked with developing the communication system of the Phenix Drone.

Implementation

Implementation

Implementation

The project is divided into two sub-sections: backend and frontend. Backend section consists of USB drone-pilot and Companion computer. USB drone pilot receives information from the sensors on drone and this information is transmitted to MavProxy inside the companion computer. USB Master inside the MavProxy receives the information and broadcasts it to the Android application through its TCP port. In the Frontend part of the app the data get fetched by a Worker-Thread and passed over to the associated views in the GUI. The user gets a notification if the values reach a critical point. For example if the battery gets low the user can land the drone as soon as possible. This is to avoid problems with the drone. Each of these values can be toggled on or off in the Flight-Overlay. The main screen is a First-Person-View from a connected camera. This is made possible by a modified USB-Video-Class (UVC)-Library made in native C++. The camera gets automatically recognized and activated once plugged in.

Evaluation and discussion

Evaluation and discussion

Although the implemented solution and methodology are pretty good and constitute a set of powerful tools to develop a mission control panel prototype for search and rescue drones. But there are still some of the provided outcomes which can be improved and extended further.

  • Develop the mobile application for all gadgets to make it platform independent.
  • Research on different Fpv Cameras to get the high-quality video streaming and to analyze the effect of the video in compressed form. Because during compression over the internet it may lose some important information.
  • The concept of implementing 3D mapping to identify the victims and current fire areas. This can be achieved by integrating the RGB and infrared channels to construct thick 3D environmental patterns.
  • Analyze the sensor values like flight time, altitude, and temperature to extend the more possible attributes and make it dynamic.
  • The user interface can be improved to display the close-to real-time video stream and to analyze the previous results in graphical form.
